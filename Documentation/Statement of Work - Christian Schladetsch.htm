Overview

This document describes a proposal to implement the Project Concept as described in the document titled 'AR_Trade_development_Christian.pdf'.

The technical requirements of the project may be broken down into four orthogonal parts:

  1. Augmented Reality camera system.
  2. Skeleton tracking and gesture recognition system.
  3. Agent-based character movement (the non-human actors in the system).
  4. Integration, deployment and testing.

The proposal is split into three main developmental phases. In the first phase, the first two key systems will be provided, integrated and tested. And in the second stage the remaining two areas will be covered, allowing then for full production to commence in phase 3.

Phase 1 - Proof of Concept

This phase would provide an application that can be run on a Windows7 computer with a camera attached.

In order to test the application, the camera must be pointed at a large area of floor with a marker. The exact representation of the marker will be provided with the application, but in general it will be something like what is shown in the reference video. Specifically, a yellow square glued to the floor approximately 5 meteres away from the camera, directly in front of it.

The testable elements of phase 1 are:

  • The application will recognise when a user enters the marker, and when the user raises a hand when standing on the marker.
  • In response to these actions, an animated 3d model - an 'agent' - (to be provided by the client) will be scripted to walk up to the user, perform an action, and then walk out of frame.
  • There will be no interaction between the agent and the user.
  • The system is only expected to work with one user visible by the system.

There are various ways this could fail:

  • Difficulty in getting a clean recognition of the skeleton at various lighting conditions.
  • The system may only have a narrow distance band where good recogition can occur.
  • Matching the vanishing point and lighting conditions used by the agents and the real scene may be difficult.

Phase 2 - Interaction

If phase 1 is a success, then we can move ahead with phase 2.

This would take as long as phase 1, and is said to be complete when:

  • All issues raised in phase 1 are addressed.
  • The system works with many people milling around the area.
  • The system correctly determines when (multiple) people walk in and out of the marker region.
  • The system provides correct triggers and target points for the agent animation system. Specifically, where to move to to interact with the user, where the user's hands are, and whatever other gestures are required as detailed in the Project Concept document.
  • Note that I will not be required to provide the actual scripting control of the agents, however I will work toward that pending art assets provided by the client.

There are various ways this could fail, including:

  • A better visual camera is needed.
  • A better depth camera is needed.
  • Skeleton tracking does not provide a high-enough resolution to properly match the agents with the users.
  • Lighting and noise may interfere with the system.
  • The system cannot track enough people at once, or fails complete when there are too many systems. Some sort of reasonable fallback state will have to be supplied in these cases.

Phase 3 - Full Production

This final phase has all the underlying systems in place and in ongoing development. This will mainly be the content production and integration phase and can continue indefinately.

Conclusion
